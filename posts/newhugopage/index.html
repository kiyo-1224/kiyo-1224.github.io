前言

此篇博客为我的第一篇博客，原本只耗费了一个小时便完成了博客的基本搭建，但因后续对主题设置的不满，便对主题进行了修改导致后续出现了一系列的问题，使得我最终花费了六个小时才完善博客的搭建

初次搭建个人博客

使用 Hugo+github 部署的方案 通过 Hugo 可以快速搭建静态网站,然后将其部署到GitHub上，使其可以在公网访问到

搭建流程

0.前置

在搭建博客之前需要先完成 git 安装与配置，并且设置好用户名和电子邮件，否则后续的步骤将无法进行

1. Hugo文件下载

在使用 hugo 之前，需要先在 Hugo 官网下载 hugo.exe 文件 地址为： https://www.gohugo.org/

2. Hugo 环境变量配置

下载完成之后，需要对环境变量进行配置，否则将无法识别 hugo指令，只需将 hugo.exe 路径加入 path 中即可完成环境变量的配置

win10 右键此电脑 -> 属性 -> 高级系统设置 -> 环境变量 -> 系统变量 -> path -> 添加即可

3.生成站点

使用 Hugo 快速生成站点

hugo new site path //path 为站点生成的路径 cd path //进入目录

站点目录结构：

archetypes/ content/ layouts/ static/      config.toml

4.创建文章

hugo 使用Markdown对内容进行管理 若要创建一个页面

hugo new pageName.md

pageName.md 将自动生成到content目录下

+++

date = “2015-10-25T08:36:54-07:00”

draft = true

title = “myPage”

+++

正文内容

+++之间的内容是TOML格式，是页面的配置，还可以使用YAML格式---标记

5.安装主题

在 Hugo 官网中的皮肤列表一栏提供了许多主题，选择一款主题，在主题的介绍中包含了其安装方法，只需执行对应的指令即可完成安装

一般的步骤为: 创建themes目录 cd themes git clone url //url 页面提供的GitHub地址

6.使用主题

在博客根目录下 使用：hugo server --theme=你的主题名 --buildDrafts 随后复制命令行中提供的地址，复制到浏览器中，在浏览器中浏览其效果 如果发生错误请检查，主题目录下的theme属性和themeDir属性名称与路径需要与其对应

7.部署

首先需要在GitHub上创建一个仓库 命名为：你的GitHub用户名.github.io 然后在主题根目录下执行 Hugo 指令生成最终界面

hugo --theme=你的主题名 --baseUrl=“url” //url为之前创建的仓库名 --buildDrafts

在一切顺利的情况下，所有的静态页面都会生成到 public 目录，将pubilc目录里所有文件 push 到刚创建的Repository的 master 分支

cd public

git init

git remote add origin https://github.com/用户名/仓库名.git

git add -A

git commit -m “提交说明”

由于GitHub不再支持密码授权，因此需要通过个人token 的方式获得授权

点击头像 -> settings -> Developer Settings -> Personal access tokens -> Genetate new token 填入说明，选择有效期 -> 点击Generate token ->复制token 使用命令： git remote set-url origin https://token @github.com/GitHub用户名/仓库名.git git push -u origin master

在一切正常的情况下，将可以使用仓库名访问到博客页面

结语

在搭建过程中并不是一帆风顺，比如最后的推送尝试许多次密码授权都未能成功，花费了许多时间才发现是Github不再支持密码授权。还有忘记进入public目录就进行关联，导致上传错误的文件，以至于最后推送访问时显示错误的页面，一些问题解决下来花费的时间，加上GitHub访问的问题，导致花费了大量的时间才完成博客的搭建
